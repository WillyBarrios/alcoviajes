# Comportamiento del agente
1. Siempre lee el archivo rules.md para entender las reglas de arquitectura y estructura de carpetas.
2. Siempre lee el archivo Design.md para entender las reglas de diseño y UI.
3. Cuando se te pida generar código, asegúrate de que el código sea limpio, organizado y bien comentado.
4. Cuando generes código, asegúrate de que el código sea modular y reutilizable.
5. Cuando generes código, asegúrate de que el código sea responsivo y se vea perfecto en móviles y escritorios.
6. Debes iterar sobre el mismo archivo hasta que sea perfecto, no pases a otro archivo hasta que el actual esté perfecto.
7. Debes usar herramientas (code run, terminal, etc) para asegurarte de que el código sea correcto y que las herramientas fallen lo menos posible.
8. No puedes usar estilos en línea, todos los estilos deben estar en el archivo styles.css o en archivos específicos de componentes.
9. No puedes usar iconos de Font Awesome, debes usar iconos de Lucide.
10. Lee el archivo commit.md para entender las reglas de commit. y ten encuenta cada una de sus reglas para hacer los commits.
11. Si no encuentras información sobre algo, debes usar las herramientas (search, code run, terminal, etc) para encontrar la información y luego continuar con el desarrollo.
12. Debes usar variables de entorno para almacenar información sensible, como claves de API, y debes asegurarte de que la información sensible no se exponga en el código.
13. Los archivos json no deben estar en la carpeta src, deben estar en la carpeta data/.
15. Los archivos .md deben estar en la carpeta agents/. Y deben ser leidos por el agente para entender las reglas de desarrollo y estructura de carpetas.
16. Los archivos .md no sben ser incluidos en los pulls, ya que son archivos de configuración para el agente.
